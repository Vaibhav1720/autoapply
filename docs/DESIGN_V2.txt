PRODUCT PILLARS
================================================================================

1. AUTH — Email/password signup + login, JWT sessions
2. PROFILE — Resume parse + manual edit (skills, experience, education,
             address, EEO answers, default cover letter)
3. COMPANY DISCOVERY — Curated list, scrape selected companies' careers pages,
                       AI-rerank against profile, return ranked job list
4. AUTOFILL — Chrome extension fills any job application form using
              rule-based mapping + AI suggestions (rules first, AI for
              uncovered/low-confidence fields, ask-panel for the rest)

================================================================================
RESUME PARSE -> PROFILE (api/function_app.py · _extract_skills_from_resume)
================================================================================

PDF text extracted with pypdf, then sent to Azure OpenAI (gpt41) with a
strict system prompt. Returns ALL of:
  firstName, lastName, skills[], education[], experience[],
  email, phone, linkedin, github, portfolio,
  address, city, state, country (full name), zip,
  summary, totalYears,
  coverLetter   <- NEW in v1.6.6: 4-6 sentence default
                  "Why are you a good fit?" answer in the candidate's voice.
                  Auto-populated into applicationDetails.coverLetter on
                  resume upload (only if user hasn't written one).

The parsed coverLetter is then used by /api/v1/autofill/suggest as the
default value for any free-text "tell us about yourself / why this role"
fields the AI smart-fill encounters.

================================================================================
MISSING-INFO LOGIC (api/function_app.py · get_missing_info)
================================================================================

REQUIRED (reported as missing if blank):
  firstName, lastName, phone,
  country, city, state, zip,
  visaStatus, willingToRelocate, salaryExpectation, noticePeriod

OPTIONAL (rendered in the editor but never flagged as missing; tracked in
_OPTIONAL_QUESTION_KEYS):
  address, githubUrl, portfolioUrl, coverLetter, remoteWork,
  gender, ethnicity, veteranStatus, disability

Completeness % is computed only over the REQUIRED set.

================================================================================
AUTOFILL SUGGEST PIPELINE (POST /api/v1/autofill/suggest)
================================================================================

For each detected field {key, label, type, options?}:
  1. MEMORY    — exact normalized-label hit in autofill_memory  -> conf 1.0
  2. HEURISTIC — regex map over label -> profile attribute
        Skips placeholder values (city='city', state='state',
        address='home', n/a, none, tbd, ...) so old junk doesn't leak.
  3. AI        — Azure OpenAI gpt41 with json_object response
        coverLetter is fed in as default for "why a good fit / tell us
        about yourself" prompts.
        Output post-processed by _validate_ai_answer:
          * _PDF_GARBAGE_RE strips MediaBox leakage like "0 0 595.32 841"
          * yes/no detector maps yes/y/true/1 -> "Yes", no/n/false/0 -> "No"
          * select forces value to one of the provided options or blanks it
  4. NONE      — confidence 0; extension shows ask-panel to user.

Anything with confidence < 0.8 also surfaces in the ask-panel; user answer
gets POSTed to /api/v1/autofill/save-answers and lives forever in memory.
Learned answers are stored in profile.applicationDetails.customAnswers as
{ normalizedKey: { label, value } } and surfaced in the options page so
the user can edit or remove any of them.
